Minimal surface area. Maximum file control.

Nodi stays focused on the workflows a file manager cannot afford to fake: upload, refresh, browse, rename, delete, download, and keep every path inside your configured root.

Durable uploads

Staged inside the target storage directory, then finalized atomically so Docker volume boundaries do not eat completed files.

Root confinement

Storage paths are resolved and checked against the configured root before file operations run.

Focused UI

List/grid views, command buttons, upload progress, modals, toasts, and dark mode without a heavy frontend framework.

Nodi uses Go and vanilla JavaScript so the application stays small enough to understand and fast enough to run on modest machines.

FAQ

Before you expose a file manager to your network.

What does GitHub Pages deploy?

Only this static landing page. The actual Nodi file manager runs on your own server or Docker host.

Can I install with Docker?

Yes. The installer pulls the container image, writes environment settings, and starts Docker Compose.

Does Nodi support normal filenames?

Yes. It supports common release and archive filenames while rejecting unsafe path and HTML-sensitive characters.

Is it secure enough for the public internet?

Use trusted network controls, HTTPS, and changed credentials before public exposure. Nodi includes session cookies, security headers, and path confinement, but deployment context still matters.
