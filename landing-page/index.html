Built for the MVP that has to work

Everything a practical file manager needs, without a heavy platform.

Nodi keeps the surface area small and the core workflows sharp: authenticated access, durable uploads, folder operations, and a UI that stays out of the way.

Private by default

Serve files from your configured root directory and keep storage on your server, NAS, VPS, or Docker volume.

Real file workflows

Create folders, upload batches, rename items, download assets, delete files, and switch views from the browser.

Fast Go backend

The server uses Go's standard HTTP stack with focused handlers and tests around the core file operations.

Docker native

Run with Docker Compose, map persistent storage, and update the app without moving your file library.

Safer paths

Root confinement, safer filename validation, session cookies, security headers, and upload-size limits are built in.

Quiet interface

Dense file tables, upload progress, theme controls, and direct actions are designed for repeated daily use.

Questions before you install.

Short answers for the operational details that matter before putting a file manager on a server.

What is Nodi for?

Nodi is for small private servers, home labs, and simple team environments where you want a browser UI over files you already control.

Does GitHub Pages host the app?

No. GitHub Pages hosts this marketing page. The Nodi app runs on your own server with Docker or the Go binary.

Can I change the port?

Yes. Set the port in the environment before running the server or installer. The current non-default local port is documented in the repo.

Is it production hardened?

The MVP has security headers, session cookies, path confinement, safer uploads, and test coverage, but you should still put it behind trusted network controls for public exposure.
